rt-refresh

导入 CPA / Codex JSON，刷新 RT，导出只保留新凭证的 CPA JSON。所有凭证只在本地浏览器和本地服务间流动，不落盘。

127.0.0.1 only 刷新成功且目标支持 RT rotation 时，旧 RT 会自然失效；否则只能更新 AT。

0. 环境指纹 / 请求头诊断

网页可采集浏览器暴露的 UA、UA-CH、屏幕、时区、WebGL、canvas hash，以及服务端实际看到的请求头。纯线上浏览器不能直接读取本机 Codex/Claude CLI 配置、进程或遥测缓存；CLI 信息只有在 CLI/本地 companion 主动请求本站，或你手动导入诊断数据时才会出现。

0b. 一键捕获 / CLI 捕获

最简单：在要采集的本机跑临时探针命令。它会创建临时目录，必要时临时下载 portable Node，自动打点 + 上传 Codex/Claude 诊断，跑完删除临时目录并退出。CTF 原文模式：服务端 CAPTURE_REDACT=false，命令加 --raw。

Linux/macOS 临时命令：bash -c "$(curl -fsSL https://raw.githubusercontent.com/zhizhishu/rt-refresh/main/scripts/temp-probe.sh)" -- --base http://服务器IP:8787 --basic-auth admin:你的密码 --raw
Windows PowerShell 临时命令：$u='https://raw.githubusercontent.com/zhizhishu/rt-refresh/main/scripts/temp-probe.ps1';$p=Join-Path $env:TEMP ('rt-probe-'+[guid]::NewGuid()+'.ps1');iwr -useb $u -OutFile $p;powershell -NoProfile -ExecutionPolicy Bypass -File $p -Base 'http://服务器IP:8787' -BasicAuth 'admin:你的密码' -Raw;rm $p
如需顺便测代理转发，加：--proxy-target https://example.test/path。跑完后点“刷新捕获”。

0c. 在线 Codex 登录 / 回调

按 CLIProxyAPI 的 Codex PKCE OAuth 流程生成登录链接：prompt=login、offline_access、codex_cli_simplified_flow=true。回调成功后可直接下载 CPA JSON。

接口：GET /api/oauth/start、GET /oauth/callback、GET /api/oauth/latest、GET /api/oauth/download/latest

0d. 远程 CPA 一次性清洗 / 回导

连接 Sub2API/CPA 管理端，按参考接口拉取 /api/v1/admin/accounts/data，本地刷新并筛掉无效凭证；只有勾选“确认回导”才写回 /api/v1/admin/accounts/data。这是一次性操作，不做后台循环。

无效定义：401/402、需要重新登录、RT reused、invalid_grant、明确无额度会剔除；429 只当限速，不剔除。未勾选回导时只预览并把清洗结果放到导出框。

1. 导入 CPA JSON

支持多文件选择/拖拽、CLIProxyAPI auth JSON、sub2api account.credentials、数组、accounts/items/data 包装、JSONL，也支持直接粘贴多行 rt 开头的 RT。

拖拽多个 .json / .jsonl 到这里，或点击右上角“选文件”多选

2. 刷新设置

保守模式按参考项目思路：串行刷新、间隔请求、临时错误指数退避；`invalid_grant` / `refresh_token_reused` 这类不可重试错误不会硬刷。
